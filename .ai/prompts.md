# Project Prompts

## P-001 Project Initialization

You are joining an ongoing Solution Architecture project for the "Arquitectura de Solución Multinube" program.

Before generating any documentation, you must understand the project context, architecture policies, repository organization and terminology.

## Step 1 – Read the AI Knowledge Base

Read and understand the following documents located in the `.ai` folder:

- context.md
- architecture.md
- glossary.md

These documents define:

- Project context
- Architecture methodology
- Documentation standards
- Repository organization
- Architecture principles
- Terminology
- AI working rules

Do not ignore them.

## Step 2 – Understand the Repository

Analyze the repository structure.

Identify:

- Enterprise Architecture documentation
- Solution Architecture documentation
- Requirements folders
- Deliverables
- Templates

Do not modify any repository structure.

---

## Step 3 – Understand the Current Deliverable

The current objective is **Deliverable 2 – Requirements Engineering**.

The objective of this deliverable is to define the requirements for each selected initiative.

Each initiative contains an Architecture Brief (README.md) which represents the primary business input.

For every initiative, the following artifacts shall exist:

- README.md
- functional-requirements.md
- non-functional-requirements.md
- acceptance-criteria.md

At this stage, only the README.md files are considered complete.

The remaining documents will be generated progressively.

---

## Step 4 – Working Rules

You shall:

- Read existing documentation before generating new content.
- Reuse repository terminology.
- Preserve document structure.
- Maintain traceability.
- Generate one artifact at a time.
- Respect the Architecture Policy defined in `.ai/architecture.md`.

You shall not:

- Invent business requirements.
- Modify approved documentation.
- Change repository organization.
- Generate additional deliverables unless requested.
